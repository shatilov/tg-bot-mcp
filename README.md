# tg-bot-mcp

MCP-сервер (Model Context Protocol) поверх [Telegram Bot API](https://core.telegram.org/bots/api): ассистент в Cursor или другом MCP-клиенте может читать входящие обновления бота, отвечать в чаты и отправлять медиа.

Транспорт: **stdio** (процесс запускается клиентом, без отдельного HTTP-сервера).

## Возможности

- **Входящие сообщения** — подтягивание апдейтов через `getUpdates`, нормализация текста, медиа, callback, опросов и др., локальное хранение в JSON.
- **Исходящие действия** — текст, фото, видео, аудио, документы, анимации, голос, стикеры; источник файла — абсолютный путь, URL или `file_id`.
- **Скачивание файлов** с Telegram по `file_id` на диск или во временную папку.

## Требования

- Node.js 18+ (нужен встроенный `fetch`)
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка и сборка

```bash
npm install
npm run build
```

Собранная точка входа: `build/index.js` (после `npm run build`).

## Переменные окружения

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `TELEGRAM_BOT_TOKEN` | да | Токен бота |
| `TELEGRAM_CHAT_ID` | нет | Числовой ID чата по умолчанию для `send_message`, `send_media`, `send_voice`, если в вызове инструмента не передан `chat_id` |
| `TELEGRAM_MCP_STORAGE_PATH` | нет | Путь к файлу состояния (JSON). По умолчанию: `~/.codex/tg-bot-mcp/storage.json` |

Скопируйте `.env.example` в `.env` и заполните значения, если запускаете вручную из shell.

## Подключение к клиенту (пример Cursor)

Пошаговая настройка для ассистента и нюансы вызова инструментов: см. [AGENTS.md](./AGENTS.md).

В настройках MCP добавьте сервер, который запускает собранный бинарь с нужными переменными окружения:

```json
{
  "mcpServers": {
    "telegram": {
      "command": "node",
      "args": ["/полный/путь/к/tg-bot-mcp/build/index.js"],
      "env": {
        "TELEGRAM_BOT_TOKEN": "ваш_токен",
        "TELEGRAM_CHAT_ID": "123456789"
      }
    }
  }
}
```

Либо установите пакет глобально / через `npx` и укажите `command`: `tg-bot-mcp` из `package.json` `bin`, если он доступен в `PATH`.

Пользователи должны **написать боту первым** (или состоять в нужном чате), чтобы появился `chat_id` для ответов.

## Инструменты MCP

| Инструмент | Назначение |
|------------|------------|
| `list_chats` | Список чатов с последним сообщением и счётчиком непрочитанного |
| `get_new_messages` | Непрочитанные апдейты (опционально фильтр `chat_id`), после выдачи помечаются прочитанными |
| `send_message` | Отправка текста |
| `send_media` | Отправка медиа заданного типа |
| `send_voice` | Отправка голосового сообщения |
| `download_telegram_file` | Скачивание файла по `file_id` |

## Ограничения и заметки

- Один процесс MCP держит одну сессию long polling через `getUpdates`. Не запускайте **два** клиента с одним и тем же токеном одновременно — апдейты будут конкурировать.
- В хранилище держится ограниченное число последних сообщений (см. константу в коде).
- Логи в stderr не мешают stdio-протоколу MCP.
