# Инструкция для агентов: подключение tg-bot-mcp

Документ для настройки MCP-клиента (Cursor, Claude Desktop и т.п.), чтобы ассистент мог вызывать инструменты Telegram через этот репозиторий.

## Что должно быть на машине

1. Репозиторий собран: из корня проекта выполнить `npm install` и `npm run build`.
2. Есть `TELEGRAM_BOT_TOKEN` от BotFather.
3. Известен числовой `chat_id`, если нужен чат по умолчанию для отправки без явного `chat_id` в каждом вызове (переменная `TELEGRAM_CHAT_ID`). Иначе перед каждой отправкой указывай `chat_id` из результата `list_chats` / `get_new_messages`.

Путь к точке входа после сборки: `build/index.js` относительно корня клона (в конфиге MCP указывай **абсолютный** путь).

## Cursor

1. Открой настройки MCP: **Cursor Settings → MCP** (или файл конфигурации MCP, если используешь project-level).
2. Добавь сервер с транспортом **stdio**: клиент запускает процесс и общается по stdin/stdout.
3. В `env` передай секреты; не вставляй токен в публичные репозитории и не коммить конфиг с реальными значениями.

Пример фрагмента конфигурации (подставь свой абсолютный путь к `build/index.js`):

```json
{
  "mcpServers": {
    "telegram": {
      "command": "node",
      "args": ["/ABS/PATH/TO/tg-bot-mcp/build/index.js"],
      "env": {
        "TELEGRAM_BOT_TOKEN": "<token>",
        "TELEGRAM_CHAT_ID": "<optional numeric chat id>"
      }
    }
  }
}
```

Опционально: `TELEGRAM_MCP_STORAGE_PATH` — путь к JSON-хранилищу; если не задан, используется путь по умолчанию из README.

После сохранения перезапусти MCP-сервер в UI Cursor или перезапусти приложение, если инструменты не появились.

## Поведение при работе с инструментами

- Сначала имеет смысл вызвать `list_chats` или `get_new_messages`, чтобы узнать актуальные `chat_id` и непрочитанное.
- `get_new_messages` помечает возвращённые записи прочитанными; вызывай осознанно.
- Для `send_message` / `send_media` / `send_voice` без `chat_id` в аргументах нужен заданный в окружении `TELEGRAM_CHAT_ID`, иначе инструмент вернёт ошибку конфигурации.
- Один и тот же бот-токен не должен одновременно обслуживаться двумя процессами с `getUpdates` (два MCP или бот + другой поллер) — апдейты будут распределяться между ними.

## Проверка без клиента

Из корня репозитория, с экспортированными переменными:

```bash
export TELEGRAM_BOT_TOKEN="..."
npm run build
node build/index.js
```

Процесс должен остаться ждать stdio; в лог stderr возможна строка вида `[mcp] server running on stdio`. Остановка: прервать процесс в терминале.
